# Version History


## Version 1.0 — Building the Foundation

### Goal

Create the core platform that collects, stores, and visualizes threat intelligence.

### Features

- Threat feed collection
- IOC normalization
- SQLite database
- Initial Power BI dashboard
- Manual threat reports


## Version 2.0 — Making Threat Data Smarter

### Goal

Enrich raw threat intelligence with context that helps analysts prioritize what matters.

### Features

- MITRE ATT&CK mapping
- CVE enrichment
- EPSS scoring
- Risk-based IOC prioritization
- Threat actor attribution


## Version 3.0 — Assisting the Analyst

### Goal

Use AI to support investigations while keeping analysts in control of security decisions.

### Features

- AI-generated incident summaries
- Executive summaries
- Investigation recommendations
- Natural language search



## Version 4.0 — Moving to the Cloud

### Goal

Deploy the platform to Azure and make it accessible through cloud-native services.

### Features

- Azure deployment
- REST API
- Authentication
- CI/CD pipeline
- Docker containers



## Version 5.0 — Production Ready

### Goal

Transform the platform into a scalable security engineering tool.

### Features

- Multiple threat intelligence sources
- Scheduled automation
- Real-time threat updates
- Performance optimization
- Production-ready deployment
